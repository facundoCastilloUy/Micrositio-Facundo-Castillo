Sobre este proyecto

Este micrositio personal forma parte del curso Diseño Web 3. El objetivo fue crear un sitio propio que funcione como carta de presentación profesional, aplicando herramientas y metodologías actuales del entorno web.

Mi enfoque y el Value Proposition Canvas

Desde el principio usé el Value Proposition Canvas como guía para definir la estrategia del sitio. En lugar de pensar solo en mostrar proyectos, busqué entender qué necesita alguien que contrata servicios de desarrollo front-end. Así llegué a la idea de ofrecer soluciones simples y claras para emprendedores o pequeñas empresas que quieren tener presencia online sin complicarse con la parte técnica.

El sitio intenta cubrir esos puntos: mostrar los servicios de forma directa, destacar la velocidad y claridad del trabajo, y tener un contacto fácil y funcional.

Contenido y decisiones de diseño

  • El botón principal “Ver planes” aparece en varios lugares para llevar al visitante directo a la acción.
  • Secciones simples con buena jerarquía visual para explicar qué hago y cómo trabajo.
  • Modo claro/oscuro para mejorar la experiencia en distintos entornos.
  • La página “Sobre mí” incluye una parte más personal y un artículo sobre el rol del diseñador digital en la era de la IA.

Herramientas y desarrollo

Todo el sitio fue hecho desde cero con HTML, Tailwind CSS y un poco de JavaScript Vanilla. Usé Netlify para publicarlo y GitHub para el control de versiones, con dos ramas (main y prod) como pedía la consigna.

  • Tailwind CSS me permitió mantener el código limpio y ordenado.
  • Formspree se usa para el formulario de contacto, ya que no tengo backend propio.
  • Los íconos se manejan con un sprite SVG para mantener el rendimiento y consistencia visual.

Aprendizajes y desafíos

Lo más desafiante fue lograr que el sitio sea funcional, limpio y escalable, sin depender de frameworks complejos. Aprendí a usar mejor Tailwind, a organizar el proyecto con Git, y a resolver la parte del formulario sin backend. También experimenté con el modo oscuro y con una estructura de código más profesional pensando a futuro.

Conclusión

Este proyecto me sirvió para poner en práctica todo lo aprendido y para sentar una base sólida como desarrollador front-end. A futuro me gustaría integrar CMS o bases de datos, pero por ahora el resultado refleja bien mi forma de trabajar: sitios claros, rápidos y pensados para que el cliente no se complique.
